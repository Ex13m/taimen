# Эвристический аудит TAIMEN — 2026-07-14 (v1.8.0)

Три независимых аудитора (архитектура · данные/автономия · документация/CI).
Ничего не правилось — только изучение. Лечение — в [SPEC-2.0.md](SPEC-2.0.md).

## Вердикт одной строкой

Для текущего масштаба структура крепкая и дисциплинированная; но
**самопроектирование стоит на одной ноге**: ночной мастер с правом автомерджа
правит 2272-строчный монолит, который CI не проверяет вообще, память — гонка
четырёх писателей без merge, а книга пожеланий — канал prompt-инъекции
с правами записи в репозиторий.

## Сильные стороны

- Дисциплина монолита: секции размечены якорями `/* ---------- */`, единый стиль,
  русские комментарии служат документацией для агента.
- Деградация везде: нет ключа → запасной мозг → честное сообщение; нет WebGL —
  нет краша; нет распознавания — кнопка прячется.
- Многоуровневый бюджет-контроль (DAILY_LIMIT / DAILY_COST_LIMIT+10% / SCOUT_DAILY_COST).
- Отдельная ветка `taimen-memory` (память не триггерит деплой), `concurrency`
  у ночного мастера, «одно пожелание за ночь».
- `window.TAIMEN` — готовый крючок для смоук-тестов.
- Тесты функций покрывают весь контракт chat.js, включая обе запаски.

## Критические находки

### К1. CI слеп к index.html
`tests.yml` гоняет только тесты функций. Синтаксическая ошибка в 2000-строчном
inline-JS проходит CI зелёной → автомердж → деплой → белый экран. Ошибка GLSL
невидима вдвойне: `initScene()` тихо возвращает null — чат работает, орба нет.
Смоук сцены существует, но живёт вне репозитория — для CI и мастера его нет.

### К2. Prompt-инъекция через книгу пожеланий
Текст `[[wish:]]` пишется из чата (без ACCESS_KEY — любым посетителем сайта)
и попадает дословно в промпт агента с правами `contents:write` +
`pull-requests:write` (night-master.yml:50). Плюс heredoc `first<<EOF` ломается
строкой «EOF» в пожелании. Это канал управления репозиторием извне.

### К3. Память: lost-update на четырёх писателях
- Браузер POST-ит **весь снимок** без read-before-write (index.html:1442–1451);
  scout, второе устройство и ночной мастер пишут тот же memory.json.
  Сценарий: страница открыта утром → scout в 10:17 добавил анализы → одна фраза
  днём → утренний снимок затёр труды разведчика **и обнулил его бюджет-счётчик**.
- Два устройства: последний пишущий побеждает целиком (merge есть только у orbits).
- Ответ сервера проглатывается: `.catch(() => {})` — 429/400/409/502 молча.
- **Бомба 64KB**: memory.js отвергает тело >64KB, а knowledge допускает до
  ~400KB — при разрастании материалов вся git-персистентность молча умирает.
- Битый JSON → `data:null` → фронт стартует с нуля → следующий persist
  перезаписывает файл пустотой.
- Дыра на закрытии вкладки: дебаунс 8с без flush на `pagehide` — «запомни X»
  и закрыл вкладку = заметка не доехала и будет затёрта git-версией.

### К4. Никогда не попадают в git (пропадут с чисткой браузера)
`history` (вся беседа), `usage`/`orbStats` (расходы), `voiceOut`. iOS Safari
выселяет localStorage неиспользуемых сайтов через 7 дней.

### К5. Daily-лимиты chat.js — декоративные
Счётчики в памяти инстанса: холодный старт обнуляет, N инстансов = N бюджетов.
Без обязательного ACCESS_KEY прохожие жгут кредиты. (У scout бюджет в git —
правильная схема, но её подрывает К3.)

### К6. Ночной мастер
- Пожелание вычёркивается из книги **до** судьбы PR: упали тесты — «выполнено» ≠ выполнено.
- «Убрать первый элемент» — если хозяин за ночь редактировал книгу, вычеркнется не то.
- Ветка `taimen-memory` не защищена: удаление/force-push → memory.js молча
  пересоздаст её пустой.

## Существенные (не критические)

- **Дубли-двойники** фронт/сервер: PRICES, потолки моделей, промпт Иммортала,
  MAPN↔ENTITIES — правка одной копии молча рассинхронизирует.
- **TDZ-мины**: `busy` (объявлен :1650, используется :1038), `t0` (:1374 ← :850) —
  перемещение блока агентом даёт ReferenceError, который тесты не ловят.
- Регекс-протокол `[[тегов]]` — неявный контракт между системным промптом и
  parseTags, ничем не проверяется.
- chat.js: `daily.count++` до вызова API (неудачи тоже съедают лимит); гонка
  на проверке бюджета; таймаут функции 10с против Fable с мышлением.
- Тесты хрупки для машинной правки: общий сценарий с накоплением состояния,
  реальные sleep 2100 — вставка кейса в середину ломает хвост.
- Мусор эволюции: мёртвый CSS `#start-unused`, пустые закомментированные блоки.
- Протокол тегов и схема memory.json нигде не документированы; CLAUDE.md нет —
  ночной мастер изучает 121KB с нуля каждую ночь.

## Автономия: что уже есть / чего нет

Есть: night-master (cron 03:17 + workflow_dispatch), scout ×3/сутки, пожелания
голосом через [[wish:]], автомердж, экран «что нового» при загрузке.
Нет: голосовой запуск цикла («обновись» — не распознаётся), работающая страница
не узнаёт о новой версии (сравнение только при загрузке), никто не предлагает
перезагрузку после мержа.
