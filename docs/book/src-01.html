02 · АУДИТ — ВЕРДИКТ ТРЁХ ГЛУБИННЫХ РЕВИЗОРОВ

Структура крепкая и дисциплинированная для монолита. Но самопроектирование стоит на одной ноге: ночной мастер с правом автомерджа правит 2 272 строки, которые CI не проверяет вообще; память — гонка четырёх писателей без слияния; книга пожеланий — открытый канал управления репозиторием.

К1 · КРИТИЧНО

CI СЛЕП К СЦЕНЕ

Ошибка в 2000-строчном JS проходит CI зелёной → автомердж → белый экран. Ошибка шейдера невидима вдвойне: чат работает, орба нет. Смоук существует, но живёт вне репозитория.

К2 · КРИТИЧНО

ИНЪЕКЦИЯ ЧЕРЕЗ КНИГУ ПОЖЕЛАНИЙ

Пожелание пишется из чата (без пароля — кем угодно) и дословно попадает в промпт агента с правами записи в репозиторий.

К5

ЛИМИТЫ ТРАТ — ДЕКОРАТИВНЫЕ

Счётчики в памяти инстанса: холодный старт обнуляет, N инстансов = N бюджетов. Без обязательного пароля прохожие жгут кредиты.

К3 · КРИТИЧНО

ПАМЯТЬ: ПОСЛЕДНИЙ ПИШУЩИЙ ЗАТИРАЕТ ВСЕХ

Браузер, второе устройство, разведчик и ночной мастер пишут один JSON целиком, без слияния. Утренний снимок затирает дневные труды Шпиона и обнуляет его бюджет. Материалы >64KB молча убивают всю git-память.

К4

БЕСЕДА ЖИВЁТ ТОЛЬКО В БРАУЗЕРЕ

История чата, расходы, статистика орбов не попадают в git — чистка браузера = амнезия. Закрыл вкладку раньше 8 секунд — заметка пропала.

К6

МАСТЕР ВЫЧЁРКИВАЕТ ДО МЕРЖА

Пожелание помечается «выполнено» до судьбы PR; ветка памяти не защищена от удаления.

ГОНКА ПАМЯТИ — КАК ТЕРЯЮТСЯ ДАННЫЕ

03 · SPEC 2.0 — СЕМЬ ЭТАПОВ К САМОПРОЕКТИРОВАНИЮ

0 · ПРЕДОХРАНИТЕЛИ

  • Парс-чек фронта в CI (~20 строк) — ловит белый экран
  • Смоук сцены в репо + required checks на main
  • Пароль ACCESS_KEY — обязателен для всех сил
  • Санитайзинг пожеланий; границы мастера (не трогает workflows и лимиты)

1 · ПАМЯТЬ-КРЕПОСТЬ

  • Слияние по полям-владельцам вместо затирания
  • Append-only журнал: из него восстановимо всё
  • Беседа и расходы — тоже в git; flush при закрытии вкладки
  • Бэкапы + детектор аномалий («было 30 заметок, стало 2 — стоп»)

2–3 · ДОКИ И СКИЛЫ

  • CLAUDE.md + 6 постоянных доков (протокол, схема памяти, карта UI)
  • Автодок на каждый пуш: DEVLOG + HISTORY со ссылками на все правки
  • Орб читает свою летопись (ссылка «летопись» в инфо-окне)
  • Скилы: release, smoke, review, night-wish, memory-edit, docs-sync

4 · САМОПРОЕКТ ПО ГОЛОСУ

/api/evolve будит мастера · подтверждение голосом · после мержа орб сам просит перегруз и перерождается

5 · МАСТЕРСКАЯ И ХУДОЖНИК

  • Вход — любые файлы: PDF, DOCX, картинки, таблицы → в знания
  • Художник: «нарисуй капитана дредноута» → картинка в чате (Replicate FLUX, ~$0.003)
  • Выход: «сделай PDF на тему…» → мозг пишет, художник рисует, PDF собирается в браузере → ссылка + исходник в память
  • Эта книга — первый образец жанра

6 · СТРУКТУРА

  • Данные из кода → config.js (сущности, цены, промпты)
  • Потом ES-модули по 150 строк — мастеру легко править

ГОРИЗОНТ — КУДА ВЕДЁТ РАЗВИТИЕ

  • Реальное время без хозяина: мастер работает очередью пожеланий, орб докладывает прогресс голосом, сам предлагает улучшения из наблюдений разведчика
  • Самоизменение личности: орб правит свои промпты и сущности через config (данные, не код — безопасный канал), рождает новых субагентов по потребности
  • Самопочинка: орб видит свои падения → сам пишет пожелание-фикс → мастер чинит ночью — петля замыкается
  • Мультиустройство: журнал событий делает честную синхронизацию телефон↔ноутбук
  • Мастерская растёт: презентации, лендинги, видео — тем же паттерном «мозг → художник → сборка»
  • Настоящий голос: tts из заглушки в живой тембр глубин
